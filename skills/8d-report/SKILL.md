# 8D 问题解决方法

## 概述

8D（Eight Disciplines）是汽车行业最广泛使用的问题解决方法，起源于福特汽车。8D方法强调先遏制问题再分析根本原因，确保问题得到彻底解决且不再发生。本技能支持完整的8D报告编写、根本原因分析和纠正措施制定。

## 何时触发

- 用户提到"8D"、"8D报告"、"问题解决"、"客诉处理"
- 需要处理客户投诉或内部质量问题
- 需要进行根本原因分析
- 供应商纠正措施要求(SCAR)

## 8D八个步骤

### D0: 准备工作
**目标**：识别问题，决定是否启动8D

**关键活动**：
1. 问题的初步识别和紧急程度评估
2. 判断是否需要启动8D（评估标准：严重度、是否重复发生、是否客户要求）
3. 获取管理层支持启动8D

### D1: 成立团队
**目标**：组建跨职能问题解决团队

**团队构成原则**：
- 团队领导：具有决策权和资源调配权
- 核心成员：涉及问题的各职能部门代表
- 扩展成员：按需加入的技术专家
- 赋予团队权限和资源

### D2: 描述问题
**目标**：清晰、准确地定义问题

**5W2H方法**：
- **What**：什么问题？什么零件？什么缺陷？
- **When**：何时发生？何时发现？是持续还是间歇？
- **Where**：在哪里发现？在哪个工序/位置？
- **Who**：谁发现的？影响哪些客户？
- **Why**：为什么这是一个问题？
- **How**：怎样发现的？缺陷如何表现？
- **How many**：多少数量？不良率多少？

**问题陈述模板**：
> 在[时间/条件]，[谁]发现[什么零件]存在[什么缺陷]，影响[数量/比例]，导致[什么后果]。

### D3: 实施临时遏制措施
**目标**：保护客户，阻止问题继续发生

**遏制措施类型**：
1. **在制品隔离**：锁定嫌疑产品，100%筛选
2. **库存排查**：仓库、运输中、客户处库存排查
3. **生产端遏制**：增加检验频次、增加防错
4. **客户端遏制**：客户处追加检查或更换

**关键原则**：
- 遏制措施必须立即生效
- 遏制范围必须覆盖所有可能受影响的产品
- 遏制措施必须验证有效
- 记录遏制措施的成本和效果

### D4: 根本原因分析
**目标**：找到问题的根本原因

**分析方法**：

#### 5Why分析法
```
问题：产品尺寸超差
Why 1：为什么尺寸超差？→ 夹具定位偏移
Why 2：为什么夹具定位偏移？→ 夹具定位销磨损
Why 3：为什么定位销磨损？→ 定位销材质硬度不足
Why 4：为什么材质硬度不足？→ 采购规格未明确硬度要求
Why 5：为什么未明确硬度要求？→ 夹具设计标准缺少耐磨件材质规范
根本原因：夹具设计标准缺少耐磨件材质规范
```

#### 鱼骨图（Ishikawa Diagram）
按6M分类分析：
- **人(Man)**：操作技能、意识、疲劳
- **机(Machine)**：设备精度、维护保养
- **料(Material)**：原材料、辅料批次变异
- **法(Method)**：工艺参数、作业标准
- **测(Measurement)**：测量方法、量具精度
- **环(Environment)**：温度、湿度、清洁度

#### 故障树分析(FTA)
自上而下演绎分析，适合复杂系统故障。

**验证根本原因**：
- 必须能够解释所有已知的故障现象
- 必须能够重现问题
- 消除根本原因后问题应不再发生

### D5: 制定永久纠正措施
**目标**：选择并验证消除根本原因的措施

**措施评估矩阵**：
| 评估维度 | 权重 | 评分(1-5) |
|---|---|---|
| 有效性（能否消除根本原因） | 高 | |
| 可行性（技术、资源） | 中 | |
| 风险（是否引入新问题） | 中 | |
| 成本 | 低 | |
| 实施周期 | 低 | |

### D6: 实施永久纠正措施
**目标**：执行纠正措施并验证有效性

**实施步骤**：
1. 制定实施计划（时间、责任人、资源）
2. 执行措施
3. 验证措施有效性
4. 更新相关文件（FMEA、控制计划、SOP等）
5. 取消临时遏制措施

### D7: 预防再发生
**目标**：防止同类问题在其他产品/过程上再发生

**预防措施**：
1. 将措施推广到类似产品/过程
2. 更新FMEA（措施输入）
3. 更新控制计划
4. 更新作业指导书
5. 更新设计/工艺标准
6. 横向展开（Yokoten）

### D8: 团队认可与关闭
**目标**：认可团队贡献，总结经验教训

**关闭活动**：
1. 验证所有措施已实施且有效
2. 确认相关文件已更新
3. 关闭8D报告
4. 团队庆祝和认可
5. 经验教训归档
