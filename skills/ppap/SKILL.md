# PPAP 生产件批准程序

## 概述

PPAP（Production Part Approval Process）是验证供应商是否正确理解了客户工程设计记录和规范的所有要求，并且在所规定的生产节拍条件下，具有持续满足这些要求的能力。PPAP是APQP第四阶段的核心输出，也是IATF 16949强制性要求。

## 何时触发

- 用户提到"PPAP"、"生产件批准"、"PPAP提交"
- 需要准备PPAP提交包
- 需要确认PPAP提交等级
- 客户要求PPAP批准时

## PPAP 18要素清单

| 序号 | 要素 | 说明 | 关键文件 |
|---|---|---|---|
| 1 | 设计记录 | 工程图样、规范、CAD数据 | 图样、3D数据 |
| 2 | 工程更改文件 | 临时更改、ECN等 | ECN单 |
| 3 | 客户工程批准 | 如需要 | 客户批准函 |
| 4 | 设计FMEA | DFMEA | DFMEA表格 |
| 5 | 过程流程图 | 制造过程流程 | 流程图 |
| 6 | 过程FMEA | PFMEA | PFMEA表格 |
| 7 | 控制计划 | 试生产/生产控制计划 | 控制计划表 |
| 8 | MSA研究 | 测量系统分析 | Gage R&R报告 |
| 9 | 尺寸结果 | 全尺寸检验结果 | 尺寸报告 |
| 10 | 材料/性能试验结果 | 材料和性能试验 | 试验报告 |
| 11 | 初始过程研究 | SPC/Cpk研究 | Cpk报告 |
| 12 | 合格实验室文件 | 实验室资质 | ISO 17025证书 |
| 13 | 外观批准报告(AAR) | 如适用 | AAR表格 |
| 14 | 样品生产件 | 代表性样品 | 实物样品 |
| 15 | 样品检查 | 主样品标识 | 检查记录 |
| 16 | 检具和试验设备 | 测量设备清单 | 设备清单 |
| 17 | 客户特殊要求 | CSR符合性 | CSR检查表 |
| 18 | 零件提交保证书(PSW) | PPAP提交总表 | PSW表 |

## PPAP提交等级

| 等级 | 提交要求 | 适用场景 |
|---|---|---|
| Level 1 | 仅PSW | 客户指定 |
| Level 2 | PSW + 样品 + 部分支持文件 | 客户指定 |
| Level 3 | PSW + 样品 + 完整支持文件 | **默认等级** |
| Level 4 | PSW + 其他客户要求 | 客户指定 |
| Level 5 | PSW + 样品 + 完整支持文件（现场评审） | 新产品/高风险 |

## PPAP提交流程

### Step 1: 确定提交等级
1. 查阅客户CSR确定提交等级
2. 默认为Level 3
3. 特殊情况由客户指定

### Step 2: 准备各要素文件
1. 按照18要素清单逐一准备
2. 确保各文件之间的一致性
3. 关键一致性检查：
   - FMEA失效模式 ↔ 控制计划特性
   - 控制计划 ↔ 作业指导书
   - 特殊特性标识在各文件中一致
   - 流程图步骤与PFMEA、控制计划对应

### Step 3: 有效生产运行
1. 使用生产设备、工装和过程
2. 生产数量：通常1-8小时产量或300件（取大值）
3. 使用生产操作人员
4. 按生产节拍运行

### Step 4: 提交与批准
1. 填写PSW（零件提交保证书）
2. 打包所有支持文件
3. 提交给客户
4. 客户批准结果：
   - **完全批准**：可按计划供货
   - **临时批准**：有限期间供货，需完成改进
   - **拒绝**：不能供货，需重新提交

## PPAP检查清单脚本

使用 `scripts/ppap_checklist.py` 生成PPAP检查清单：
```bash
python3 scripts/ppap_checklist.py --level <1-5> --customer <OEM名称> --output <output_path>
```
