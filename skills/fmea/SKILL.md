# FMEA 失效模式与影响分析

## 概述

FMEA（Failure Mode and Effects Analysis）是汽车行业最核心的预防性质量工具。2019年发布的AIAG-VDA协调版FMEA采用七步法，以行动优先级(AP)替代了传统的RPN。本技能支持DFMEA（设计FMEA）和PFMEA（过程FMEA）的全流程分析。

## 何时触发

- 用户提到"FMEA"、"DFMEA"、"PFMEA"、"失效模式分析"
- 需要进行风险评估或失效模式识别
- 需要生成或更新FMEA表格
- 控制计划需要FMEA输入时

## AIAG-VDA FMEA七步法

### 步骤1：策划与准备（Planning and Preparation）
**目标**：确定FMEA项目范围和团队

**关键活动**：
1. 确定FMEA类型（DFMEA/PFMEA/FMEA-MSR）
2. 组建跨职能团队（设计、工艺、质量、生产等）
3. 定义分析范围（边界图、接口分析）
4. 收集基础信息（以往FMEA、保修数据、客诉数据）
5. 建立FMEA时间线

**输出**：FMEA策划表、团队名单、项目范围定义

### 步骤2：结构分析（Structure Analysis）
**目标**：将分析对象分解为系统/子系统/组件层级

**DFMEA结构分析**：
- 系统 → 子系统 → 组件 → 零件
- 使用边界图识别系统接口

**PFMEA结构分析**：
- 过程 → 过程步骤 → 过程子步骤 → 过程要素（4M：人/机/料/法）
- 使用过程流程图作为输入

### 步骤3：功能分析（Function Analysis）
**目标**：明确每个结构元素的功能及要求

**关键活动**：
1. 描述每个结构元素的预期功能
2. 将功能与要求关联
3. 识别功能间的依赖关系
4. 标注特殊特性（安全/法规相关）

### 步骤4：失效分析（Failure Analysis）
**目标**：识别潜在失效模式、失效影响和失效原因

**分析链路**：
- 失效影响（FE）→ 失效模式（FM）→ 失效原因（FC）
- 每个功能对应可能的失效模式
- 每个失效模式对应可能的失效影响
- 每个失效模式对应可能的失效原因

**常用失效模式类别**：
- DFMEA：功能丧失、功能退化、间歇性功能、非预期功能、功能超时
- PFMEA：过程缺陷、过程变异过大、过程遗漏、过程多余操作

### 步骤5：风险分析（Risk Analysis）
**目标**：评估当前控制措施的有效性，确定行动优先级

**评估维度**：
- **严重度(S)**：失效影响的严重程度（1-10分）
- **频度(O)**：失效原因发生的频率（1-10分）
- **探测度(D)**：当前探测措施的有效性（1-10分）

**行动优先级(AP)**（替代RPN）：
- **H（高）**：需要立即行动
- **M（中）**：需要行动，可按优先级排序
- **L（低）**：可考虑行动，但非必须

**AP判定逻辑**（基于S-O-D组合查表）：
```
如果 S=9-10 且 O≥4 → AP=H
如果 S=9-10 且 O=2-3 且 D≥5 → AP=H
如果 S=9-10 且 O=2-3 且 D≤4 → AP=M
如果 S=6-8 且 O≥6 且 D≥4 → AP=H
如果 S=6-8 且 O≥5 且 D≤3 → AP=M
...（完整查表见参考文档）
```

### 步骤6：优化（Optimization）
**目标**：针对高/中AP项目制定改进措施

**措施类型**：
- **预防措施**：降低频度(O) — 修改设计/过程、增加防错
- **探测措施**：降低探测度(D) — 增加探测方法、改进探测手段
- **严重度(S)**：通常无法通过措施降低，除非改变设计

**关键原则**：
- 优先考虑预防措施（降低O），而非仅增加探测（降低D）
- 措施应具体、可执行、有责任人、有完成时间
- 措施实施后需重新评估S-O-D和AP

### 步骤7：结果文件化（Documentation of Results）
**目标**：形成FMEA报告，与其他文件关联

**输出文件**：
- FMEA表格（DFMEA/PFMEA）
- 措施跟踪表
- 与控制计划的关联矩阵
- 特殊特性清单更新
- 经验教训数据库更新

## FMEA与控制计划的关联

FMEA输出直接输入到控制计划：
- PFMEA中的失效模式 → 控制计划中的"产品/过程特性"
- PFMEA中的控制措施 → 控制计划中的"控制方法"
- PFMEA中的特殊特性 → 控制计划中的"特殊特性分类"
- PFMEA中的探测方法 → 控制计划中的"评价/测量技术"
