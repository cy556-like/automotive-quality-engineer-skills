# DMAIC 六西格玛改进

## 概述

DMAIC是六西格玛的核心改进方法论，由五个阶段组成：Define（定义）、Measure（测量）、Analyze（分析）、Improve（改进）、Control（控制）。本技能支持完整的DMAIC改进项目策划与执行，帮助质量工程师系统性地解决复杂质量问题。

## 何时触发

- 用户提到"DMAIC"、"六西格玛"、"改善项目"、"6σ"
- 需要开展数据驱动的质量改进项目
- 需要进行根本原因的统计分析
- 过程改进需要系统方法论

## DMAIC五个阶段

### D: Define（定义阶段）
**目标**：明确改进项目的范围、目标和团队

**关键输出**：
- 项目章程（Project Charter）
- SIPOC图（供应商-输入-过程-输出-客户）
- 客户呼声(VOC)分析
- 关键质量特性(CTQ)树
- 项目时间线

**项目章程要素**：
| 要素 | 说明 |
|---|---|
| 问题陈述 | 当前状况的客观描述（含数据） |
| 目标陈述 | 改进后的期望状态（SMART原则） |
| 项目范围 | 包含和不包含的边界 |
| 业务案例 | 为什么需要改进（财务影响） |
| 团队成员 | 黑带、绿带、过程所有者等 |
| 时间计划 | 各阶段预计完成时间 |

### M: Measure（测量阶段）
**目标**：量化当前过程表现，建立基线

**关键输出**：
- 过程基线数据（Cpk、DPMO、不良率）
- 测量系统分析（MSA）确认
- 数据采集计划
- 过程流程图详解
- 因果矩阵（C&E Matrix）

**数据采集原则**：
- 确保测量系统可靠（先完成MSA）
- 采集足够的数据量（通常≥30个数据点）
- 区分长期和短期数据
- 记录数据采集条件

### A: Analyze（分析阶段）
**目标**：识别影响过程输出的关键因素（Vital Few）

**关键工具**：
| 工具 | 用途 | 适用场景 |
|---|---|---|
| 假设检验 | 验证因素影响的显著性 | 比较两组或多组数据 |
| 方差分析(ANOVA) | 多因素影响分析 | 多个因素同时影响 |
| 回归分析 | 建立输入-输出关系 | 连续型数据关系 |
| 相关性分析 | 变量间关系强度 | 初步因素筛选 |
| 多变异图 | 变异来源分析 | 区分位置/时间/件间变异 |
| 排列图 | 关键少数识别 | 缺陷类型分布 |

**常用假设检验方法**：
- 正态性检验：Anderson-Darling、Shapiro-Wilk
- 均值比较：1-Sample t、2-Sample t、Paired t
- 方差比较：F-test、Levene's test
- 比率比较：1-Proportion、2-Proportion
- 非参数方法：Mann-Whitney、Kruskal-Wallis

### I: Improve（改进阶段）
**目标**：制定并验证改进方案

**关键输出**：
- 改进方案（基于根本原因）
- 试验设计(DOE)结果
- 改进效果验证
- 风险评估（FMEA更新）
- 试运行计划

**试验设计(DOE)**：
| DOE类型 | 适用场景 | 因素数 |
|---|---|---|
| 全因子设计 | 因素较少时 | 2-4 |
| 部分因子设计 | 因素较多时 | 5-15 |
| 响应曲面(RSM) | 寻找最优参数 | 2-5 |
| 田口方法 | 稳健参数设计 | 2-10 |

### C: Control（控制阶段）
**目标**：维持改进成果，防止回退

**关键输出**：
- 更新后的控制计划
- SPC监控计划
- 标准化作业文件
- 过程管理制度
- 项目收益核算
- 经验教训总结

**控制策略**：
1. 统计监控（SPC控制图）
2. 防错(Poka-Yoke)措施
3. 可视化管理
4. 标准化作业
5. 定期审核

## 六西格玛关键指标

| 指标 | 公式/定义 | 目标 |
|---|---|---|
| DPMO | 百万机会缺陷数 | ≤3.4 |
| Sigma水平 | 基于DPMO查表 | ≥4.5 |
| COPQ | 质量成本/营收 | 持续降低 |
| RTY | 各工序合格率乘积 | 接近100% |
