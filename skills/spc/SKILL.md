# SPC 统计过程控制

## 概述

SPC（Statistical Process Control）是汽车行业核心质量工具之一，通过统计方法监控过程稳定性，评估过程能力，及时发现过程异常。本技能支持各类控制图的生成与分析、过程能力指数的计算与评估，以及SPC在PPAP提交中的应用。

## 何时触发

- 用户提到"SPC"、"控制图"、"过程能力"、"Cpk"、"Ppk"
- 需要进行过程稳定性分析
- PPAP提交需要初始过程能力研究
- 需要识别过程异常模式

## 控制图类型选择

### 计量型数据控制图
| 控制图 | 适用场景 | 子组大小 |
|---|---|---|
| Xbar-R（均值-极差） | 最常用，子组较小时 | 2-9 |
| Xbar-S（均值-标准差） | 子组较大时 | ≥10 |
| I-MR（单值-移动极差） | 每次只能取一个数据 | 1 |

### 计数型数据控制图
| 控制图 | 适用场景 | 数据类型 |
|---|---|---|
| P图（不合格品率） | 子组大小可变 | 不合格品率 |
| NP图（不合格品数） | 子组大小固定 | 不合格品数 |
| C图（缺陷数） | 子组大小固定 | 缺陷数 |
| U图（单位缺陷数） | 子组大小可变 | 单位缺陷数 |

## 过程能力分析

### 短期过程能力（Cp/Cpk）
- **Cp**（过程能力指数）= (USL - LSL) / (6σ_within)
- **Cpk**（过程能力性能指数）= min[(USL - X̄) / (3σ_within), (X̄ - LSL) / (3σ_within)]
- PPAP要求：Cpk ≥ 1.33（特殊特性）或 Cpk ≥ 1.67（安全特性）

### 长期过程性能（Pp/Ppk）
- **Pp**（过程性能指数）= (USL - LSL) / (6σ_overall)
- **Ppk**（过程性能性能指数）= min[(USL - X̄) / (3σ_overall), (X̄ - LSL) / (3σ_overall)]

### 能力评估标准
| 等级 | Cpk值 | 过程状态 |
|---|---|---|
| 优秀 | ≥2.0 | 过程高度受控 |
| 良好 | 1.67-2.0 | 过程受控 |
| 合格 | 1.33-1.67 | 满足PPAP要求 |
| 需改进 | 1.0-1.33 | 不满足特殊特性要求 |
| 不合格 | <1.0 | 过程能力不足 |

## 异常模式识别（Nelson Rules）

在控制图上识别以下异常模式：
1. **规则1**：1个点超出3σ控制限
2. **规则2**：连续9个点在中心线同一侧
3. **规则3**：连续6个点递增或递减
4. **规则4**：连续14个点上下交替
5. **规则5**：连续3个点中有2个在2σ之外
6. **规则6**：连续5个点中有4个在1σ之外
7. **规则7**：连续15个点在1σ之内（层化）
8. **规则8**：连续8个点在1σ之外（混合模式）

## SPC分析脚本

使用 `scripts/spc_analysis.py` 进行SPC分析：
```bash
python3 scripts/spc_analysis.py --data <data_file.csv> --chart-type <Xbar-R|I-MR|P|C> --usl <值> --lsl <值> --output <output_path>
```
